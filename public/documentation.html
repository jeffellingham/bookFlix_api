neXup Documentation Page

The neXup app will initially be a web application that provides users with information about various movies, directors, actors, genres, and more. Users will be able to sign up, update their personal information, and create a list of their favorite movies from the site's selection.

The dream for the future of neXup is for it to become an invaluable resource for users to catalog, rate, and rank every movie and show they've watched and be able to use the app to find recommendations on what to watch next.

URL endpoints

Business Logic URL HTTP Method Request body format Response body format
Get list of all movies /movies GET None A JSON object holding data about all the movies, including their title, description, director, genre, an image URL, featured or not, release year, and actors.
Get data on single movie /movies/:title GET None A JSON object holding data about a single movie, contains description, genre, director, image URL, featured or not, release year, and actors.
Get all movies with an actor /movies/:actor GET None A JSON object holding data about a one or more movies, containing description, genre, director, image URL, featured or not, release year, and actors.
Get data on genre by name /movies/genre/:genreName GET None A JSON object containing a brief description of each genre: { "name": "Action", "description": "Should contain numerous scenes where action is spectacular and usually destructive. Often includes non-stop motion, high energy physical stunts, chases, battles, and destructive crises (floods, explosions, natural disasters, fires, etc.)." }
Get data on director by name /movies/directors/:directorName GET None A JSON object with data on the director, contains bio, birth year, death year (if applicable), and a list of their best movies. Example: { "name": "Hayao Miyazaki", "bio": "Hayao Miyazaki is a Japanese animator, filmmaker, and manga artist.", "born": "1941" }
Create account for a new user /users POST A JSON object holding data about the user that's to be added/registered, structured like: { "name": "JimBob", "favoriteMovies": ["Snatch", "Gladiator"] } A JSON object holding data about the student that was registered: { "name": "JimBob", "favoriteMovies": ["Snatch", "Gladiator"], "id": "8e65a264-5ba7-4885-8ecb-f6948a7030b3" }
Update user's account info /users/:id PUT A JSON object containing the user's new name, structured like: { "name": "Kayla" } A JSON object containing the user's account info: { "id": 1, "name": "Kayla", "favoriteMovies": [ "Die Hard", "Eternal Sunshine of the Spotless Mind"] }
Update user's favorite movie list /users/:id/:movieTitle POST None A text message confirming whether the new favorite movie was added.
Get user's favorites list /users/:id/favorites GET None A JSON object containing all of the user's favorite movies: [ "Iron Man", "The Dark Knight" ]
Delete movie from user's favorite list /users/:id/:movieTitle DELETE None A text message confirming whether the former favorite movie was removed.
Delete user account (deregister) /users/:id DELETE None A text message confirming whether the user's account was deleted.
